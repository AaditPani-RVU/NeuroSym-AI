Guard — The Core Engine

Guard engine/guard.py

The central engine that routes input through rules and returns a structured result. Operates in three modes: information-first (offline), LLM-first (generate + repair loop), and streaming.

Constructor

from neurosym import Guard

guard = Guard(
    rules=[...],          # list of Rule instances
    llm=my_llm,           # optional — needed for .generate() / .stream()
    max_retries=2,        # LLM repair loop attempts
    deny_above="high",    # auto-block violations ≥ this severity
    deny_rule_ids=[...],  # specific rule IDs that always hard-deny
    enable_offline_repairs=True,
)

Information-First API (no LLM required)

.apply_text(text, meta?)→ GuardResult — evaluate raw string
.apply_json(obj, meta?)→ GuardResult — evaluate dict/list
.apply(Artifact)→ GuardResult — evaluate any Artifact
.aapply_text(text)→ async GuardResult — rules run concurrently via asyncio
.aapply_json(obj)→ async GuardResult

LLM-First API

.generate(prompt, **kwargs)→ GuardResult — generate + validate + repair loop
.stream(prompt, **kwargs)→ Generator[str, GuardResult] — yield chunks, abort on hard-deny
.agenerate(prompt)→ async GuardResult — LLM in executor, rules concurrent
Offline repairs: Guard automatically strips Markdown code fences and extracts embedded JSON blocks before validation — no LLM needed for these cleanups.

Artifact

Artifact frozen dataclass

Universal container for any information entering the pipeline. Guards work on Artifacts, not raw strings — this is what "information-first" means.

fieldtypemeaning
kindstr"text" or "json" — determines how rules interpret content
contentAnyThe actual data (str for text, dict/list for json)
metadict?Optional context: trace_id, origin, timestamp, etc.
artifact = Artifact(kind="text", content="raw voice transcription", meta={"source": "mic"})

GuardResult

GuardResult dataclass

Returned by every Guard call. Contains the full audit trace — what was checked, what fired, and what was repaired.

fieldtypemeaning
okboolTrue iff zero violations — always the top-level pass/fail
blockedboolAlways not ok — the allow/deny decision
hard_deniedboolTrue iff a violation matched deny_above or deny_rule_ids
violationslist[dict]All violations: {rule_id, severity, message, meta}
repairslist[dict]Offline deterministic repairs applied
near_misseslist[dict]Borderline signals from NearMissRule implementors
tracelist[TraceEntry]Full per-attempt audit log
artifactArtifact?The repaired artifact that was validated
result = guard.apply_text("some input")
result.ok            # True / False
result.blocked       # True if any violation
result.hard_denied   # True only if severity ≥ deny_above
result.user_summary() # safe-to-show violations (no attack spans)
result.to_dict()     # full JSON-serializable trace

Violation & Severity

Violation frozen dataclass

Produced by a rule when it detects a problem. Contains a stable rule ID, human-readable message, severity level, optional machine-readable meta, and a user-safe message.

fieldtypemeaning
rule_idstrStable dotted identifier e.g. adv.prompt_injection
messagestrFull audit message (may contain matched text — never show users)
severitySeverityOne of the five levels below
metadict?Machine-readable context: matched spans, paths, scores
user_messagestr?Sanitized message safe to display to end users

Severity Levels

info low medium high critical

Ordered: info < low < medium < high < critical. deny_above="high" blocks high and critical violations.

Rule System — Base

Rule Protocol

The minimal interface every rule satisfies. Just two things: a stable string id and an evaluate(output) → list[Violation] method. An empty list means the rule passed.

BaseRule Optional base class

Convenience base that provides fail(), ok(), async aevaluate(), and a template method pattern. Override check() instead of evaluate().

class MyRule(BaseRule):
    id = "biz.no_competitors"

    def check(self, output):
        if "acme" in output.lower():
            return self.fail("mentioned competitor", severity="medium")
@rule decorator One-liner authoring

Turns a plain generator function into a Rule instance without subclassing. Yield violation message strings; return nothing to pass.

from neurosym.rules.base import rule

@rule(id="biz.no_competitors", severity="medium")
def no_competitor_mentions(text):
    for c in ["acme", "umbrella"]:
        if c.lower() in text.lower():
            yield f"mentioned competitor: {c}"
StreamingRule Protocol

Rules implementing this protocol are evaluated incrementally during Guard.stream(). A hard-deny from a streaming rule stops chunk emission immediately — the triggering chunk is never forwarded to the consumer.

feed(chunk: str)→ list[Violation] — process one arriving chunk
finalize()→ list[Violation] — stream ended, emit any remaining violations
reset()→ None — clear state before a new stream begins
SecretLeakageRule implements StreamingRule with a 256-char overlap buffer so credentials split across chunk boundaries are still caught.
NearMissRule Protocol

Optional protocol for rules that can report borderline signals — inputs that passed but were close to being blocked. Guard calls near_miss() only on passing results and returns them in GuardResult.near_misses. Useful for threshold tuning and human review flagging.

result = guard.apply_text(borderline_input)
if result.ok and result.near_misses:
    flag_for_review(result.near_misses)  # [{rule_id, score, message}]

Input Rules

PromptInjectionRule Input

Detects adversarial attempts to override LLM instructions using 9 preset attack categories derived from known taxonomies (PAIR, DAN, GCG, Perez et al. 2022, Garak corpus). Sub-millisecond regex matching.

Attack Presets

ignore_instructions role_switch system_commands path_traversal delimiter_injection exfiltration obfuscation indirect_injection system_prompt_extraction 
rule = PromptInjectionRule()                              # all presets
rule = PromptInjectionRule(presets=["ignore_instructions"]) # specific only
rule = PromptInjectionRule(extra_patterns=[r"my_pattern"]) # custom on top
rule = PromptInjectionRule(severity="critical")            # override severity

The pack system (injection-v1.json) allows loading versioned pattern sets identified by SHA-256 fingerprint for auditability.

BanTopicsRule Input

Detects what is being requested, not how. Catches clinically or academically worded synthesis requests that score near 0.0 on tone/injection models. Input is canonicalized: zero-width chars, Cyrillic homoglyphs, and spaced letters are all neutralized before matching.

Topic Presets

cbrn_weapons drug_synthesis self_harm_methods malware_exploit 
rule = BanTopicsRule()                                  # all presets
rule = BanTopicsRule(presets=["cbrn_weapons"])           # specific
rule = BanTopicsRule(extra_patterns=[r"\bmy_topic\b"])   # extend
Bidirectional patterns: "synthesize TATP" and "TATP synthesis" both fire. Negative lookaheads suppress false positives on defensive-security queries.
SemanticInjectionRule Input pip install neurosym-ai[embeddings]

Embedding-based fallback that catches paraphrase attacks bypassing the regex layer. Uses sentence-transformers to compute cosine similarity against a curated anchor set of injection examples. ~20ms latency.

guard = Guard(rules=[
    PromptInjectionRule(),   # fast regex — sub-ms
    SemanticInjectionRule(), # semantic fallback — ~20ms
])

The tail_fraction=0.25 default evaluates the last 25% of input independently alongside the full text — catches attacks where a long innocent preamble dilutes the full-text similarity score below threshold.

IntentClassifierRule Input new in v0.4 pip install neurosym-ai[classifier]

Zero-shot NLI intent detection using facebook/bart-large-mnli on CPU — no GPU required. Catches harmful intent expressed in novel phrasing that regex rules miss. ~400 MB model download on first use.

rule = IntentClassifierRule(
    bad_intents=["weapons synthesis", "jailbreak attempt"],
    threshold=0.75,   # confidence to trigger (0–1)
    severity="high",
)
result.violations[0]["meta"]["label"]  # matched intent label
result.violations[0]["meta"]["score"]  # confidence score 0–1

Output Rules

SecretLeakageRule Output StreamingRule

Detects credentials and secrets in LLM responses. Implements StreamingRule — catches secrets split across chunk boundaries via a 256-char overlap window. Triggers on:

AWS access key IDs GitHub tokens Google API keys Slack tokens Stripe keys PEM private keys JWTs Bearer tokens Passwords in URLs Generic api_key=... 
rule = SecretLeakageRule(severity="critical", max_examples=3)
SystemPromptRegurgitationRule Output

Detects when an LLM echoes your system prompt back to the user verbatim — a common prompt extraction attack. Initialize with the system prompt text you want to protect.

rule = SystemPromptRegurgitationRule(
    system_prompt="You are a helpful assistant. Instructions: ..."
)

ActionPolicyRule — Agent Plans

ActionPolicyRule Input

Validates structured JSON action plans before an agent executes them. A plan is a dict with an intent, steps[], and optional requires_confirmation.

Pre-built Factories

destructive_needs_confirmation()delete/move/format require requires_confirmation=true
no_high_risk_without_intent()send_email/upload require a non-empty intent field
max_steps(n)cap plan length to prevent runaway agents
no_path_outside_sandbox(allowed_paths)block path traversal in step parameters
from neurosym.rules.action_policy import (
    destructive_needs_confirmation, max_steps, no_path_outside_sandbox
)

# Custom policy via lambda
rule = ActionPolicyRule(
    id="policy.no_network_at_night",
    policy=lambda plan: not (
        any(s["action"] == "open_url" for s in plan.get("steps", []))
        and is_night_time()
    ),
    message="Network actions blocked during off-hours.",
    severity="high",
)

Utility Rules

RegexRule Either

Pattern-based text validation. Fails when a pattern matches (deny-mode) or when a required pattern is absent.

RegexRule(
    id="fmt.phone",
    pattern=r"\d{3}-\d{4}",
    match_means="pass"
)
SchemaRule Either

JSON Schema enforcement for structured outputs. Validates against a jsonschema dict.

SchemaRule(
    id="schema.invoice",
    schema={"required": ["amount"]},
)
DenyIfContains Either

Banned substring detection. Simple, fast, and explicit. Good first line for known bad strings.

DenyIfContains(
    id="ban.weapons",
    banned=["explosive", "weapon"],
)
PythonPredicateRule Either

Arbitrary Python predicate as a rule. For logic that doesn't fit regex or schema.

PythonPredicateRule(
    id="biz.word_count",
    predicate=lambda t: len(t.split()) < 500,
    message="Response too long",
)

Policy Algebra — Composite Rules

Combine any rules with boolean logic to express complex policies as readable, testable Python expressions.

AllOf
A AND B AND C…
Collects violations from all children. Fails if any child fails. Use to group related rules under one ID.
AnyOf
A OR B OR C…
Fails only if ALL children fail (passes if at least one passes). Use when any of several conditions is sufficient to allow.
Not
NOT A
Inverts a rule — turns a "pass" into a "fail" and vice versa. Use to express absence conditions.
Implies
A → B
Fails only when the antecedent passes but the consequent fails. "If risky, must have confirmation."
from neurosym.rules.composite import AllOf, AnyOf, Not, Implies

# Block if BOTH injection detected AND action is destructive without confirmation
combined = AllOf([
    PromptInjectionRule(presets=["ignore_instructions"]),
    Not(destructive_needs_confirmation()),
], id="compound_threat")

ConversationGuard — Multi-Turn

ConversationGuard new in v0.4 engine/conversation.py

Standard Guard is stateless — each call sees only what you pass it. ConversationGuard keeps a rolling session of prior turns so rules evaluate full conversation context. This catches slow-burn escalation attacks and cross-turn information leakage.

Constructor

cg = ConversationGuard(
    rules=[DenyIfContains(...), MyConvRule()],
    window=20,        # last N turns; 0 = unlimited history
    deny_above="high"  # forwarded to inner Guard
)

Session API

with cg.session() as s:
    s.add("user", "Let's discuss chemistry.")      # record without evaluating
    s.add("assistant", "Sure, what aspect?")
    result = s.check("user", "How do I make an explosive?")
    result.blocked  # True — full history included in context

# Async version
async with cg.asession() as s:
    result = await s.acheck("user", "...")

# Survive request boundaries — serialize and restore
state = s.state()   # JSON-serializable dict
with cg.session(restore_state=state) as s:
    ...              # session resumes from saved history
Thread-safe: add() and check() use a threading.Lock so multiple coroutines or threads can call them concurrently. The async path uses asyncio.Lock and never blocks the event loop.
ConversationRule Protocol new in v0.4.3

Optional protocol for rules that need structured turn access — per-role logic, ordering checks, or multi-turn pattern matching that flat text concatenation would lose.

class UserOnlyRule:
    id = "conv.user-only"

    def evaluate_turns(self, turns: list[Turn]) -> list[Violation]:
        user_text = " ".join(t.content for t in turns if t.role == "user")
        if "forbidden" in user_text:
            return [Violation(rule_id=self.id, message="...")]

ConversationGuard routes a rule to evaluate_turns() if it implements ConversationRule, or evaluate() if it only implements Rule. A rule implementing both is routed to evaluate_turns() only — no double-evaluation.

Benchmark Harness

BenchmarkRunner / BenchmarkCase / BenchmarkResult neurosym.bench

Built-in adversarial benchmark with 134 labeled prompt injection cases (104 attacks, 30 safe). Run against your guard configuration to measure block rate, false positive rate, and per-category performance.

from neurosym.bench import BenchmarkRunner, BenchmarkCase

cases = BenchmarkCase.load_builtin("prompt_injection")  # 134 cases
results = BenchmarkRunner(guard).run(cases)

results.block_rate          # fraction of attacks blocked
results.false_positive_rate # fraction of safe inputs wrongly blocked
results.avg_latency_ms
results.by_category()       # dict[category → BenchmarkResult]
print(results.report())      # formatted table to stdout
Baseline results — PromptInjectionRule default: 79.8% block rate, 0.0% false positives, 0.48ms avg latency.

LangChain Adapter

NeurosymCallbackHandler new in v0.4 pip install neurosym-ai[langchain]

Drop into any LangChain chain to enforce input and output guards without changing chain logic. Guards fire at LLM call boundaries; raise_on_violation=True raises ValueError, False records silently.

handler = NeurosymCallbackHandler(
    input_guard=Guard(rules=[PromptInjectionRule()], deny_above="high"),
    output_guard=Guard(rules=[SecretLeakageRule()],  deny_above="critical"),
    raise_on_violation=True,
)
llm = ChatOpenAI(callbacks=[handler])
handler.last_input_result   # GuardResult from last input check
handler.last_output_result  # GuardResult from last output check

SAT / SMT Policy Linter

PolicyLinter (Z3-backed) neurosym.policy.sat pip install neurosym-ai[z3]

Translates composite rule algebra into Z3 boolean formulas and checks for logical defects. Catches policy bugs before deployment, not at runtime.

issue kindmeaning
unsatisfiablePolicy can never pass — would block everything
tautologyPolicy always passes — dead check, never blocks
subsumedOne rule makes another redundant in the same AllOf

Agent System

Agent Loader neurosym.agents

Loads agent system prompts from .md files on disk. Prompts are cached after first read. Ship custom agents by placing my_agent.md in neurosym/agents/ or a custom directory.

from neurosym.agents import get_agent, list_agents

list_agents()               # ['neurosym_dev_agent', 'security_auditor']
prompt = get_agent("neurosym_dev_agent")   # str, cached

Failures raise typed exceptions (AgentNotFoundError, AgentLoadError) so errors are never silent.

ImpactForecaster neurosym.agents.impact_forecaster

Agent that maps code changes to blast radius: analyzes architecture maps, codeowners, and GitHub diffs to forecast which components a change affects and by how much.
